# API 명세

REST API 및 Socket Event 명세 입니다.

<br>

```
두 개의 호스트 중 하나로 이용해 주시면 감사합니다.

배포 서버: https://goodganglabs.xyz
로컬 서버: http://localhost
```

<br>

## 주의사항

- API 요청/응답 Parameter 또는 Body 내 데이터 구조는 엄격히 지켜집니다.
  - 예시에 존재하지 않는 데이터 또는 타입을 요청할 경우 400 Bad Request가 응답 됩니다.
- Error Response 시 해당 API의 Error 명세를 확인해 주세요.
- Socket Event 명세의 경우, Client 기준에서 작성되어 있습니다.
  - ex) Emit - Client에서 socket.emit()을 호출할 때

## REST API

### GET /auth/code - sessionId 유효성 검증

#### Request

```SHELL
# Header
{
    "Cookie": "sessionid=123adsf2347h9234kjh123"
}
```

#### Response

```SHELL
200 OK
# Body
{
    "status": 200,
    "message": "ok"
}
```

<br>

```SHELL
401 UnAuthorized
# Body
{
    "statusCode": 401,
    "message": "인증정보가 유효하지 않습니다."
}
```

<br>

### POST /auth/code - sessionId 발급

#### Request

```SHELL
# Body
{
    "code": "서버에 설정된 환경변수 ENTRY_CODE 입력"
}
```

#### Response

```SHELL
200 OK
# Header
{
    "Set-Cookie": "sessionId=4aa2d624beb56475b07e2ef7f1232b8ff; Max-Age=86400; Path=/; Expires=Sat, 28 Jan 2023 01:55:27 GMT; Secure"
}

# Body
{
    "status": 200,
    "message": "ok"
}
```

<br>

```SHELL
401 UnAuthorized
# Body
{
    "statusCode": 401,
    "message": "인증정보가 유효하지 않습니다."
}
```
